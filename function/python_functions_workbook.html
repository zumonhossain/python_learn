📘 Concept 1 — Normal Function

No parameters, no arguments — just a named task

The simplest function takes no input. It performs a fixed task every time it is called. Think of it as a saved recipe — you run it when you need the result.

🏥

📗 Concept 2 — Positional Parameters

Order matters — inputs mapped by position

Positional parameters receive values in the exact order they are passed. The first argument fills the first parameter, the second fills the second, and so on. Most everyday functions work this way.

🚗
Transport / Ride-hailing
Calculate Taxi Fare
Positional Params ✅ Solved

A taxi app calculates fare from distance (km) and rate per km. Both values change per ride — they must be passed as arguments.

taxi_fare.py
Positional 
def calculate_fare(distance_km, rate_per_km):
    """
    Calculate taxi fare.
    Args:
        distance_km  : trip distance in kilometres
        rate_per_km  : charge per kilometre (¥)
    """
    base_fare = 30                       # ¥30 minimum flag-fall
    fare = base_fare + (distance_km * rate_per_km)
    print(f"Distance : {distance_km} km")
    print(f"Rate     : ¥{rate_per_km}/km")
    print(f"Total    : ¥{fare:.2f}")

# City cab: 12 km at ¥15/km
calculate_fare(12, 15)

# Airport cab: 35 km at ¥20/km
calculate_fare(35, 20)
Key point — order matters: calculate_fare(12, 15) means distance=12, rate=15. If you wrote calculate_fare(15, 12) you'd get the wrong answer. Python fills positional params left to right.
▶ Input
calculate_fare(12, 15)
◀ Output
Distance : 12 km
Rate : ¥15/km
Total : ¥210.00
▶ Input
calculate_fare(35, 20)
◀ Output
Distance : 35 km
Rate : ¥20/km
Total : ¥730.00

📙 Concept 3 — Default Parameters

Smart fallbacks — optional arguments with preset values

Default parameters let you call a function with fewer arguments when the common case is predictable. The caller can always override them when needed.

📧
HR / Corporate
Employee Onboarding Email Generator
Default Params ✅ Solved

The HR system sends a welcome email to every new employee. Most join as "Junior" in the "General" department — but some have different roles and departments that override the defaults.

onboarding.py
Default Params 
def send_welcome_email(name, role="Junior Associate", department="General"):
    """
    Generate a welcome email.
    Default role and department suit 80% of new hires;
    senior or specialist hires override as needed.
    """
    print(f"To      : {name}")
    print(f"Role    : {role}")
    print(f"Dept    : {department}")
    print(f"Message : Welcome aboard, {name}! "
           f"You join us as {role} in {department}.")

# Standard hire — only name needed
send_welcome_email("Riya Sharma")

send_welcome_email("Arjun Mehta", role="Senior Engineer")

send_welcome_email("Priya Patel", role="Lead Data Scientist",

Why default params? Most hires share the same role and department. Writing send_welcome_email("Riya") is cleaner than forcing every caller to repeat "Junior Associate", "General" every time. Defaults encode the most common case.

📓 Concept 4 — *args (Variable Positional Arguments)

Accept any number of values — collected as a tuple

*args lets a function accept any number of positional arguments. Python packs them into a tuple inside the function. Use it when you don't know in advance how many values the caller will supply.

🍕
Restaurant / Food Delivery
Restaurant Order Bill Calculator
*args ✅ Solved

A restaurant table can order 1 item or 20. The billing function must handle any number of dish prices — a perfect case for *args.

restaurant_bill.py
*args 
def calculate_bill(*dish_prices):
    """
    Sum any number of dish prices into a final bill.
    *dish_prices collects all positional args as a tuple.
    """
    print("── Order Summary ──")
    for i, price in enumerate(dish_prices, 1):
        print(f"  Item {i}: ¥{price}")
    subtotal = sum(dish_prices)
    gst      = round(subtotal * 0.05, 2)   # 5% Tax on food
    total    = round(subtotal + gst, 2)
    print(f"  Subtotal : ¥{subtotal}")
    print(f"  Tax (5%) : ¥{gst}")
    print(f"  TOTAL    : ¥{total}")

# Table of 2 — 3 dishes
calculate_bill(250, 180, 120)

print()

# Table of 5 — 6 dishes
calculate_bill(350, 200, 150, 90, 400, 60)
How *args works: calculate_bill(250, 180, 120) → inside the function, dish_prices becomes the tuple (250, 180, 120). No matter how many prices are passed, the function handles them all with one sum(). You can loop, index, or pass it to any built-in that accepts iterables.
▶ Input
calculate_bill(250, 180, 120)
◀ Output
── Order Summary ──
  Item 1: ¥250
  Item 2: ¥180
  Item 3: ¥120
  Subtotal : ¥550
  Tax (5%) : ¥27.5
  TOTAL : ¥577.5

def shipping_cost(*weights_kg):
    """Calculate shipping cost for any number of packages."""
    total_weight = sum(weights_kg)
    rate = 60 if total_weight <= 5 else 50   # bulk discount
    cost = total_weight * rate

📔 Concept 5 — **kwargs (Variable Keyword Arguments)

Named flexibility — collected as a dictionary

**kwargs collects any number of named arguments into a dictionary. Use it when the caller needs to supply different fields for different situations — like form data or configuration options.

🏨
Hospitality / Hotel
Hotel Room Booking Form
**kwargs ✅ Solved

A hotel booking system collects guest details. Different bookings have different optional fields: some guests add meal plans, some request early check-in, some add spa packages. **kwargs handles them all without pre-defining every possible option.

hotel_booking.py
**kwargs 
def book_room(guest_name, room_type, **extras):
    """
    Confirm a hotel booking.
    guest_name, room_type : required positional args
    **extras              : any additional preferences the
                            guest requests (meal plan, view,
                            early check-in, etc.)
    """
    print(f"╔══ Booking Confirmation ══╗")
    print(f"  Guest     : {guest_name}")
    print(f"  Room Type : {room_type}")
    if extras:
        print("  Extras:")
        for key, value in extras.items():
            print(f"    • {key.replace('_',' ').title()}: {value}")
    print(f"╚═════════════════════════╝")
    print()

# Simple booking
book_room("Anjali Singh", "Deluxe Double")

# Booking with extras
book_room(
    "Raj Kapoor", "Suite",
    meal_plan="Full Board",
    sea_view=True,
    early_check_in="10:00 AM",
    spa_package="Couple Massage"
)
How **kwargs works: Any keyword arguments beyond the named parameters get packed into a dictionary called extras. So book_room("Raj", "Suite", meal_plan="Full Board", sea_view=True) produces extras = {"meal_plan": "Full Board", "sea_view": True}. You can iterate it, check keys, or store it in a database — it's just a dict.
▶ Input
book_room("Raj Kapoor", "Suite",
  meal_plan="Full Board",
  sea_view=True,
  early_check_in="10:00 AM")
◀ Output
╔══ Booking Confirmation ══╗
  Guest : Raj Kapoor
  Room Type : Suite
  Extras:
    • Meal Plan: Full Board
    • Sea View: True
    • Early Check In: 10:00 AM

📕 Concept 6 — Return Values

Send results back — use the output, don't just print it

A function that only prints is a dead end — you can't use its result in another calculation. return sends a value back so the caller can store it, pass it on, or make decisions with it.

🏦
Banking / Finance
Interest Calculator (Simple & Compound)
Return Values ✅ Solved

A banking app calculates interest earned. The result must be returned so other parts of the app — a report, a tax calculator, a comparison screen — can use it.

interest.py
Return Values 
def simple_interest(principal, rate, years):
    """Return simple interest earned."""
    return round(principal * rate * years / 100, 2)

def compound_interest(principal, rate, years, n=12):
    """
    Return compound interest earned.
    n = number of compounding periods per year (default: monthly)
    """
    amount = principal * (1 + rate / (100 * n)) ** (n * years)
    return round(amount - principal, 2)

def compare_interest(principal, rate, years):
    """Use returned values to build a comparison report."""
    si = simple_interest(principal, rate, years)     # ← uses return
    ci = compound_interest(principal, rate, years)    # ← uses return
    print(f"Principal        : ¥{principal:,}")
    print(f"Rate             : {rate}% p.a.  | Years: {years}")
    print(f"Simple Interest  : ¥{si:,}")
    print(f"Compound Interest: ¥{ci:,}")
    print(f"Difference       : ¥{round(ci-si,2):,} more with compounding")

compare_interest(100_000, 8, 5)
Why return instead of print? compare_interest() calls both helper functions and uses their returned values to compute the difference. If simple_interest only printed, there'd be no way to subtract from it. Return when the result needs to be used. Print when it only needs to be shown.
▶ Input
compare_interest(100_000, 8, 5)
◀ Output
Principal : ¥1,00,000
Rate : 8% p.a. | Years: 5
Simple Interest : ¥40,000.00
Compound Interest: ¥48,985.97
Difference : ¥8,985.97 more with compounding

Education
Return Multiple Values — Exam Result Card
Return Values ✅ Solved

Python can return multiple values as a tuple — perfect for an exam result that needs to give back percentage, grade, and pass/fail status all at once.

exam_result.py
Return Values 
def evaluate_result(marks_obtained, total_marks=100):
    """
    Returns:
        percentage  : float
        grade       : str  (A+, A, B, C, D, F)
        status      : str  (Pass / Fail)
    """
    pct = round((marks_obtained / total_marks) * 100, 2)

    if   pct >= 90: grade = "A+"
    elif pct >= 80: grade = "A"
    elif pct >= 70: grade = "B"
    elif pct >= 60: grade = "C"
    elif pct >= 40: grade = "D"
    else:          grade = "F"

    status = "Pass" if pct >= 40 else "Fail"
    return pct, grade, status    # ← returns a tuple of 3 values

# Unpack the 3 returned values
percentage, grade, status = evaluate_result(82, 100)
print(f"Marks: 82/100 | {percentage}% | Grade: {grade} | {status}")

# The returned values can be used in further logic
if status == "Pass" and grade in ("A+", "A"):
    print("🏅 Eligible for merit scholarship!")
▶ Input
evaluate_result(82, 100)
◀ Output
Marks: 82/100 | 82.0% | Grade: A | Pass
🏅 Eligible for merit scholarship!
▶ Input
evaluate_result(35, 100)
◀ Output
Marks: 35/100 | 35.0% | Grade: F | Fail
