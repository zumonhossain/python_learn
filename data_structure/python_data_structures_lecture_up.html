What Are Data Structures?

Different containers for storing data — each with its own rules and superpowers.

Why does it matter?

Imagine organising your room 🏠

You wouldn't pile everything in one heap. You'd use a numbered shelf for books (ordered), a sealed box for valuables (can't change), a jar where every item is unique (no duplicates), or a labelled drawer where each slot has a name (key-value).

Python's four main data structures work exactly the same way. Picking the right one makes your code faster, cleaner, and harder to break.

📋
List
[ ]
Ordered, changeable, allows duplicates. Like a shopping list.
📦
Tuple
( )
Ordered, unchangeable, allows duplicates. Like a sealed package.
🔵
Set
set( )
Unordered, unique items only. Like a ballot box.
📚
Dictionary
{key: val}
Key-value pairs. Like a real dictionary.

Python Lists

Ordered, changeable, allows duplicates — the most used structure in Python.

Concept 1 · What is a List?

A list stores multiple items in one variable, in order

Think of a list like a numbered shopping list. Each item has a position (called an index) starting at 0. You can freely add, remove, and change items.

🛒

Real life: Shopping cart

Your online cart is a list — items appear in order, you can add "Milk" twice, and remove "Bread" anytime. That's exactly a Python list.

# Square brackets [ ] create a list
cart = ["milk", "bread", "eggs", "milk"]  # duplicates OK!
print(cart)
print(len(cart))  # 4
['milk', 'bread', 'eggs', 'milk'] → 4

✅ Yes
Changeable
✅ Yes

Concept 2 · Accessing Items

Get any item using its index number (starts at 0!)

Every item has a position starting from 0. Use negative numbers to count from the end — -1 is always the last item.

🎬

Real life: Cinema seats

Seat 0 is the first, seat 1 is second. seats[-1] is the last seat without needing to count how many there are!

fruits = ["apple", "banana", "cherry", "mango"]
print(fruits[0])    # "apple"  ← first
print(fruits[-1])   # "mango"  ← last
print(fruits[1:3])  # ["banana","cherry"] ← slice
💡
Counting always starts at 0. The 1st item is [0], the 2nd is [1]. This trips up almost every beginner!
🎵 Playlist

playlist[0] is the song playing now. playlist[-1] is the last song. Skip to track 3 with playlist[2].

Concept 3 · Changing Items

Update any item by assigning a new value to its index

fruits = ["apple", "banana", "cherry"]
fruits[1] = "orange"  # replace banana
print(fruits)  # ["apple", "orange", "cherry"]
📝 To-do app

Realise task 2 was wrong? tasks[1] = "Buy birthday cake". The list updates instantly.

Concept 4 · Adding Items

Three ways to add items to a list

fruits = ["apple", "banana"]

fruits.append("cherry")    # add to END — most common
print(fruits)  # ["apple","banana","cherry"]

fruits.insert(1, "mango")   # add at position 1
print(fruits)  # ["apple","mango","banana","cherry"]

more = ["grape", "kiwi"]
fruits.extend(more)           # add ALL items from another list
🛍️ Online order

Keep adding to cart: cart.append("headphones"). Merge two wishlists: cart.extend(wishlist).

Concept 5 · Removing Items

Several ways to delete items

fruits = ["apple", "banana", "cherry", "banana"]

fruits.remove("banana")  # removes FIRST match
fruits.pop(0)             # removes by index, returns it
del fruits[0]             # delete by index
fruits.clear()            # wipe everything → []
🎵 Playlist

Remove a song: playlist.remove("Song"). Skip current song: playlist.pop(0).

Concept 6 · Looping Through a List

Use a for loop to process every item

students = ["Alice", "Bob", "Carol"]

for student in students:
    print(f"Hello, {student}!")

# With index number
for i, student in enumerate(students):
    print(f"{i+1}. {student}")  # 1. Alice  2. Bob  3. Carol
📧 Send emails

Loop through subscribers and send each one an email: for email in subscribers: send(email).

Concept 7 · List Comprehension

Build a new list in one clean line — the Pythonic way

Instead of a loop + append, write it all in a single line inside square brackets. Beginners find this mind-blowing once it clicks!

# Old way (3 lines)
squares = []
for x in range(1, 6):
    squares.append(x**2)

# List comprehension (1 line!)
squares = [x**2 for x in range(1, 6)]
print(squares)  # [1, 4, 9, 16, 25]

# With a filter — only even numbers
evens = [x for x in range(1, 11) if x % 2 == 0]
print(evens)    # [2, 4, 6, 8, 10]

# Transform — uppercase all names
names = ["alice", "bob"]
upper = [n.upper() for n in names]  # ["ALICE","BOB"]
🏪 Filter products

Get items under ₹500 from a price list: cheap = [p for p in prices if p < 500]. One line instead of six!

Concept 8 · Sorting a List

Arrange items in alphabetical or numerical order

nums   = [3, 1, 4, 1, 5, 9]
fruits = ["banana", "apple", "cherry"]

nums.sort()               # sorts IN PLACE → [1,1,3,4,5,9]
nums.sort(reverse=True)  # descending → [9,5,4,3,1,1]

# sorted() keeps original unchanged, returns new list
result = sorted(fruits)
print(result)   # ["apple","banana","cherry"]
print(fruits)   # ["banana","apple","cherry"] ← unchanged!
🏆 Leaderboard

Sort scores highest first: scores.sort(reverse=True). The winner is scores[0].

Python Tuples

Ordered and unchangeable. Once sealed, it stays sealed forever.

Concept 1 · What is a Tuple?

A tuple is like a list that can never be changed

Once created, items are locked in — you can't add, remove, or change them. This is called being immutable. It sounds limiting, but it's actually a feature that protects important data.

🎂

Real life: Birth certificate

Your name, birthday, and birthplace never change. A tuple is perfect for data like this — GPS coordinates, an RGB colour, a date — things that are facts, not variables.

# Round brackets ( ) create a tuple
location = (35.6762, 139.6503)  # Tokyo coordinates
colour   = (255, 165, 0)        # Orange in RGB
print(type(location))  # <class 'tuple'>
⚠️
Single-item tuple needs a trailing comma! (42,) is a tuple. (42) is just the number 42. The comma is what makes it a tuple.
Ordered
✅ Yes
Changeable
❌ No
Duplicates
✅ Yes
Syntax
( )
Concept 2 · Accessing Items

Access by index — exactly like a list

colour = (255, 165, 0)  # Red, Green, Blue
print(colour[0])    # 255 — Red
print(colour[-1])   # 0   — Blue (last)
print(colour[0:2])  # (255, 165) — slicing works too

if 255 in colour:
    print("Max red!")
🗺️ Map coordinates

Get latitude with tokyo[0] and longitude with tokyo[1]. These are facts — a tuple protects them from accidental changes.

Concept 3 · Unpacking a Tuple

Assign each value to its own variable in one line

One of Python's most loved features. Instead of [0], [1], [2] — name each value directly:

colour = (255, 165, 0)
red, green, blue = colour       # unpack!
print(red, green, blue)        # 255  165  0

# Swap two variables using tuple magic
a, b = 10, 20
a, b = b, a                    # a=20, b=10 — no temp variable needed!

# Functions return multiple values as a tuple
def get_size():
    return 1920, 1080          # returns a tuple
width, height = get_size()
print(f"{width}x{height}")    # 1920x1080
🎨 Design app

A colour picker returns (255, 165, 0). Unpack it: r, g, b = colour_picker() and apply each channel to your CSS.

Concept 4 · Why Use a Tuple Over a List?

Three solid reasons to choose a tuple

  • Safety: Prevents accidental changes to important data (coordinates, settings, credentials)
  • Speed: Tuples are faster than lists — Python optimises them because it knows they never change
  • Dict keys: Tuples can be used as dictionary keys — lists cannot
# Trying to change a tuple — it will ERROR on purpose!
coords = (35.6762, 139.6503)
coords[0] = 0
# ❌ TypeError: 'tuple' object does not support item assignment

# Tuple as a dictionary key (lists CAN'T do this!)
city_map = {(35.68, 139.69): "Tokyo", (48.85, 2.35): "Paris"}
print(city_map[(35.68, 139.69)])  # "Tokyo"
Reference · Tuple Methods

Tuples have only 2 methods — because they're immutable!

MethodWhat it doesExample

✅

Python Sets

Unique items only, unordered. Built for speed and maths-style operations.

Concept 1 · What is a Set?

A set keeps only unique items — duplicates vanish automatically

🗳️

Real life: Ballot box

Each person votes once. Even if someone tries to vote twice, only one vote counts. Python sets work the same — drop in duplicates and they're quietly ignored.

# Curly braces { } with values = set
colours = {"red", "blue", "green"}

# Duplicates removed automatically!
tags = {"python", "coding", "python", "tutorial", "coding"}
print(tags)  # {'python', 'coding', 'tutorial'} — 3 unique items

# Convert a list → set to deduplicate instantly
votes = ["Alice", "Bob", "Alice", "Carol", "Bob"]
unique = set(votes)
print(unique)  # {'Alice', 'Bob', 'Carol'}
🚨
{} creates an empty DICTIONARY, not a set! For an empty set you must write set(). This is the #1 beginner gotcha with sets.
Ordered
❌ No
Changeable
✅ Yes
Duplicates
❌ No
Syntax
set( )
Concept 2 · Adding & Removing Items

Modify a set — but you can't access items by index

Sets are unordered, so there's no "first" or "second" item. You can't do my_set[0]. But you can add/remove by value.

fruits = {"apple", "banana"}

fruits.add("cherry")       # add one item
fruits.update(["mango", "grape"])  # add multiple

fruits.remove("banana")    # error if missing!
fruits.discard("pineapple") # safe — no error if missing ✓

# Membership check — blazing fast even with millions of items!
if "apple" in fruits:
    print("Found it!")
Concept 3 · Set Operations — The Superpower

Sets can do Venn diagram maths: union, intersection, difference

A = {1, 2, 3, 4, 5}
B = {4, 5, 6, 7, 8}

A | B   # Union — ALL unique items from both   → {1,2,3,4,5,6,7,8}
A & B   # Intersection — ONLY in BOTH          → {4, 5}
A - B   # Difference — in A but NOT in B       → {1, 2, 3}
A ^ B   # Symmetric diff — in one but not both → {1,2,3,6,7,8}
👥 Social network — mutual friends

Find who Alice and Bob both follow: mutual = alice_follows & bob_follows. Instagram literally does this for "People you might know". One line of Python!

🔐 App permissions

What features does Pro unlock over Free? pro_only = pro_features - free_features. Show users exactly what they're upgrading for.

Python Dictionaries

Store data as key-value pairs. Look up any value instantly using its name.

Concept 1 · What is a Dictionary?

A dictionary maps a key to a value — like a real dictionary

📖

Real life: A real dictionary

Look up "apple" (key) → find "a round fruit" (value). In Python, my_dict["name"] gives the value stored under that key. Each key must be unique.

student = {
    "name":   "Aiko",
    "age":    22,
    "grade":  "A",
    "passed": True
}
print(student)
print(len(student))  # 4 key-value pairs
Ordered
✅ (3.7+)
Changeable
✅ Yes
Dup keys
❌ No
Syntax
{k: v}
Concept 2 · Accessing Items

Get values by key — two ways (one is much safer!)

student = {"name": "Aiko", "age": 22}

# Method 1: [ ] — crashes with KeyError if key missing
print(student["name"])           # "Aiko"

# Method 2: .get() — returns None or default if missing ✓
print(student.get("score", "N/A"))  # "N/A" — no crash!

# Get all keys, values, or pairs
student.keys()    # dict_keys(["name", "age"])
student.values()  # dict_values(["Aiko", 22])
student.items()   # dict_items([("name","Aiko"),("age",22)])
💡
Always prefer .get() when you're not 100% sure the key exists. It prevents crashes in real apps.
Concept 3 · Adding & Changing Items

Same syntax for both adding a new key and updating an existing one

student = {"name": "Aiko", "age": 22}

student["grade"] = "A"    # ADD new key
student["age"]   = 23    # UPDATE existing key

# update() — add or change multiple pairs at once
student.update({"city": "Tokyo", "grade": "A+"})
👤 User profile

User updates their bio: profile["bio"] = "New bio". User changes email: profile["email"] = "new@email.com". Same syntax for add and update!

Concept 4 · Removing Items

Four ways to delete entries

student = {"name": "Aiko", "age": 22, "grade": "A"}

removed = student.pop("grade")  # remove key, return value → "A"
del student["age"]               # delete by key
student.popitem()                 # remove last inserted pair
student.clear()                   # wipe everything → {}
Concept 5 · Looping Through a Dictionary

Loop over keys, values, or both at the same time

student = {"name": "Aiko", "age": 22, "grade": "A"}

for key in student:               # loop over keys
    print(key)                    # name, age, grade

for value in student.values():    # loop over values
    print(value)                  # Aiko, 22, A

for key, value in student.items():  # ← MOST USEFUL!
    print(f"{key}: {value}")
🧾 Receipt printer

Loop over an order dict and print each item + price: for item, price in order.items(): print(f"{item}: ₹{price}").

Concept 6 · Nested Dictionaries

Dicts can contain other dicts — perfect for real-world data

Almost all real data is nested. A school has students. Each student has details. A dict inside a dict handles this perfectly.

school = {
    "s1": {"name": "Alice", "grade": "A"},
    "s2": {"name": "Bob",   "grade": "B"},
}
# Access nested values with chained keys
print(school["s1"]["name"])   # "Alice"

# Loop through all students
for sid, info in school.items():
    print(f"{info['name']} got {info['grade']}")
🌐 JSON from an API

Every API response (weather, products, social media) comes back as nested JSON. Python loads it directly into a nested dictionary. This is how the real world works!

Decision Guide

Ask yourself one question — get your answer

📋

Use LIST when…

Order matters AND you need to freely add, remove, or change items. Good for: playlists, shopping carts, search results, to-do lists.

📦

Use TUPLE when…

Data is fixed and must never change. Good for: coordinates, RGB colours, database rows, returning multiple values from a function.

🔵

Use SET when…

Uniqueness matters OR you need ultra-fast membership checks. Good for: tags, unique visitors, email blocklists, mutual-friend finding.

📚

Use DICTIONARY when…

Each piece of data needs a meaningful name. Good for: user profiles, product details, config settings, counting word frequency.

Common Beginner Mistakes
🚨
{} is a DICT, not a SET. Use set() for an empty set. This catches almost every beginner at least once.
⚠️
Single-item tuple needs a comma: (42,) is a tuple. (42) is just the number 42. The comma makes it a tuple.
⚠️
Lists can't be dictionary keys. d[[1,2]]="x" crashes. Use a tuple: d[(1,2)]="x".
⚠️
Sets have no guaranteed order. Don't rely on the order items come out. Use sorted(my_set) if you need order.
Handy Conversions
# Deduplicate a list (order not preserved)
unique = list(set(["a","b","a","c"]))

# Deduplicate, preserve order (Python 3.7+)
unique = list(dict.fromkeys(["b","a","b","c"]))  # ["b","a","c"]

# list ↔ tuple
tuple([1,2,3])  # (1,2,3)
list((1,2,3))   # [1,2,3]

# dict → list of keys / values / pairs
d = {"a":1, "b":2}
list(d)           # ["a","b"]
list(d.values())  # [1,2]
list(d.items())   # [("a",1),("b",2)]
